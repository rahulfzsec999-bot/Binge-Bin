Watch List — Feature Index

A running summary of everything built into watchlist.html so far.

Core list

  • Add titles with type, genre, platform, source, and where-seen tags
  • Tabs: All / To Watch / Watched / Deleted, with live counts and a sliding red pill indicator
  • Search (in the "⋮" menu) filters the current tab by title
  • Sort: Custom order / Newest added / A–Z / Platform (in the "⋮" menu)
  • Swipe left on an item → reveals bin, tap to confirm delete
  • Swipe right on an item → reveals pencil, tap to open the edit form
  • Deleted tab: swipe (either direction) → permanently deletes, no recovery
  • Undo toast after every delete
  • Duplicate title warning with "Add anyway" option

Organizing

  • Drag handle (left dots) — tap and hold to reorder, with auto-scroll near screen edges
  • Long-press anywhere on an item → enters batch selection mode (bar at bottom: mark watched / delete / cancel)
  • Edge-swipe (from far left/right of screen) switches tabs, with a sliding page animation

Movie/show lookup (TMDB)

  • Live autocomplete suggestions under the "Title you just saw..." field (3 visible, scrollable)
  • Tap a suggestion → fills title, type, poster, year
  • Auto-detects which streaming service it's on in Australia and fills the platform dropdown
  • Tap an item to expand: description, runtime/season count, dedicated "Watch Now" button (opens the native app if installed and it has a public URL scheme, else the web page/search)

Fun extras

  • Shake the phone (5 genuine direction reversals, not just one jolt) → "Surprise me" vertical picker-wheel animation, lands on a random To Watch item
  • One-time popup after 1 minute in the app introducing the shake-to-pick feature, with a shaking popcorn icon

Personalization

  • Splash screen: red "W" animation + sound, morphs into the pinned header
  • Red/white/black theme throughout
  • Landscape mode: list left, add-form right (form fixed, not independently scrollable)
  • Real app icon (red W on black) baked into the HTML — works properly with "Add to Home Screen"
  • Rating (👍/👎) and watched-date tracking, editable per item

Data

  • Stored in localStorage — private to each device, not shared even if the link is shared
  • Export/Import as JSON backup (in the "⋮" menu)

Known limitations (platform, not fixable in code)

  • iOS Safari doesn't support the vibration/haptics API — none of the haptic calls in the code actually do anything on iPhone
  • Stan, Binge, Paramount+, Max, SBS On Demand have no public app URL scheme, so "Watch Now" opens those in the browser even if the app is installed
  • TMDB API key lives in plain text in the HTML file (fine for personal use, visible to anyone with the file)

Setup reminders

  • TMDB API key is already filled in
  • Re-upload to tiiny.host after every change, and open a fresh tab (not cached) to see updates
